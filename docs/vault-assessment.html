📖 _Vault Guide.md — Operating Rules

Routing Rule
Sort by what attention a note needs: dated→Areas/Journal, active-with-finish-line→Projects, ongoing-life→Areas, reference→Resources, done→Archive.
Domain-Hub Shape
4 standardized sections: What's true right now · What these notes mean (pattern across them) · How it connects (cross-hub links) · Next steps (concrete actions).
Project Standard
Skeleton: hub note + Tasks Board.md + .base views. Tasks are task-notes on Kanban (not checkboxes). Status vocab: inbox·active·someday·done·dead.
ADR Immutability
Decisions are immutable once accepted. Supersede with a new ADR, never rewrite history. Template: type, status, date, decision, summary, affected areas.
Agent Rule
Never alter Ben's original text. Agents may add frontmatter, links, and structure — but the prose content of notes is untouchable.
Maintenance
Nightly auto-commit + conformance check. Root sweep files loose captures. Detection only — does NOT auto-mutate content. Stale docs describe internal PARA layout; live vault uses Johnny-Decimal.
